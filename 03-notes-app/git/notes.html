Config Git

config file
  • /etc/gitconfig file: Contains values for every user on the system and all their repositories. If you pass the option --system to git config, it reads and writes from this file specifically.
  • ~/.gitconfig file: Specific to your user. You can make Git read and write to this file specifically by passing the --global option.
    $ git config --global user.name "John Doe" 
    $ git config --global user.email johndoe@example.com
  • config file in the git directory (that is, .git/config) of whatever repository you’re currently using: Specific to that single repository.

Each level overrides values in the previous level

Check config

 $ git config --list

Getting and Creating Projects

init

$ git init // use git init to make an existing directory of content into a new Git repository. You can do this in any directory at any time, completely locally.

clone

$ git clone git://github.com/schacon/simplegit.git // use git clone to get a local copy of a Git repository so you can look at it or start modifying it.

Basic Snapshotting

add

$ git add hello.rb // add specific file

$ git add . // add all files in working directory

status

$ git status // check changed files

$ git status -s // with the -s option, which gives you short output.

diff

$ git diff // show diff of unstaged changes

$ git diff --cached // show diff of staged changes

$ git diff HEAD // show diff of all staged or unstaged changes

$ git diff --stat // show summary of changes instead of a full diff

commit

$ git commit -m 'commit message' // commit files with commit message

$ git commit -am 'commit message' // automatically stage all tracked, modified files before the commit

reset

$ git reset HEAD -- file name with extension // remove specific file from staging (reverts the staging to what was)

rm, mv

$ git rm file // remove specific file from staging (kicks the file off the stage entirely)

$ git rm file // remove specific file from staging and also delete from hard disc

Branching and Merging

branch

$ git branch // list your available branches

$ git branch testing // create a branch name is testing

$ git branch -d testing // delete testing branch

$ git checkout -b erlang // create a branch called "erlang" and switch to immedicatly

checkout

$ git checkout master // switch to master branch

merge

$ git merge removals // merge branch "removals" into current branch

log

$ git log // show commit history of a branch

$ git log --oneline // short output

$ git log --oneline --graph // show merge and commit in graph

$ git log --oneline erlang ^master // see the commits in master branch

tag

$ git tag -a v0.9 558151a // add book mark for "v0.9" to specific commit "558151a"

Sharing and Updating Projects

fetch, pull

$ git fetch github // after can you can see all change for that

$ git merge github/master // merge data from master into your current branch

pull

$ git pull origin develop // do fetch and merge from develop into current branch

push

$ git push origin develop // upload data to remote

remote

list your remote aliases

$ git remote // list your remote aliases

$ git remote -v // show actual URL for each alias.

add a new remote repository of your project

$ git remote add github git@github.com:schacon/hw.git //

removing an existing remote alias

$ git remote rm origin
